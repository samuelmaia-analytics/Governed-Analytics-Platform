# Recruiter Summary

## 60-second overview

This repository is not only a dashboard.
It is an Analytics Engineering case with governance controls, privacy-aware publication, and executive delivery.

## What is technically strong

- modular pipeline with reproducible local execution;
- clear internal vs published data boundary;
- LGPD-inspired classification and explainable risk score;
- quality checks and publication decision states;
- test/lint/type-check gates in CI.

## What is simulated (explicitly)

- legal governance metadata (controller/operator/DPO) with fictional values;
- mini RIPD document for demonstration;
- enterprise security and IAM stack integration.

## Interview talking points

1. Why separating curated and published layers reduces risk and confusion.
2. How publication status is determined from privacy + quality signals.
3. Trade-offs between analytical utility and data minimization.
4. How this foundation could be promoted to production in a real company.

## Role alignment

- Data Analyst / Senior Data Analyst: KPI semantics, quality evidence, executive storytelling.
- Analytics Engineer: contracts, transformations, semantic layer, governance gates.
- Data Engineer: pipeline orchestration, artifact materialization, data reliability.
- Data Governance / Data Platform: control lifecycle, risk rationale, publication policies.
